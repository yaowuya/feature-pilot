# FeaturePilot 1.0.0 Release Notes

发布日期：2026-07-23

## 概览

`1.0.0` 将 FeaturePilot 收敛为一套可在 Claude Code 与 Codex 间复用的、证据优先的功能开发流程。重点是降低初始化与默认执行成本，同时为需要隔离执行、增量审查和视觉验收的任务提供可追溯的 SDD 证据链。

## 主要更新

### 最小信息层与动态上下文

- 新项目默认只创建 `fp-docs/manifest.md`；settings、`intel/project-facts.md`、`unknowns.md` 和 `decisions.md` 均按明确批准懒创建。
- 已有信息层通过 `.freshness.json` 的 source fingerprint/body hash 实时识别 `project-facts.md` 的 stale/conflict，不持久化 stale verdict，也不批量覆盖人工内容。
- `fp-execute-sdd` 从 manifest、相关 settings、可选 project facts、当前 change、当前源码/config 和搜索候选动态组装 task context；静态 `sdd-handoff.md` 缺失不再单独阻塞执行。

### 执行、审查与证据

- `fp-execute` 保持简洁直接，按已确认计划完成任务。
- `fp-execute-sdd` 为每个 task/final scope 固定 review scope 与最多三次审查。第三次后只允许可复现的非核心视觉差异记录为 debt；主流程问题保持 BLOCKER，不开启无限复审。
- `fp-review` 引入 Scope Matrix、跨变更 owner 识别、命令安全分类、增量 evidence 与 phase-aware dispatch/resume 检查。
- 最终审查包持久化完整视觉 case schema、provenance、evidence channels 和固定 `PASS | FAIL | CANNOT_VERIFY` verdict。

### Figma 与真实运行时视觉验收

- Figma MCP 可用时，指定节点必须在实现前调用 `get_design_context`；不可用时必须提供已批准设计源或记录 blocker，禁止伪造上下文。
- 视觉 case 统一记录设计源、Figma 节点与修订、运行时 route/state、viewport/DPR/locale/theme、fixture、reference/current/diff、mask、acceptance rule 与 replay command。
- `reference.png` 只来自已批准的 Figma/static design source；`current.png` 只来自真实目标运行时。核心 source/runtime 缺失为 `CANNOT_VERIFY` 主流程阻塞。

### CodeGraph 与双运行时

- CodeGraph 仍是可选加速层；自动安装仅使用 `npm install -g @colbymchenry/codegraph@latest`，失败会回退到原有搜索。
- 源码写入后会标记 graph 为 dirty；执行类流程返回前对已有图最多执行一次 post-write sync。
- Claude Code 与 Codex 保持同一基础版本和共享 skills。仓库本地 `sync-plugin-runtimes` skill 可验证源码、插件源和两个运行时缓存一致性。

## 升级注意事项

- `fp-docs/manifest.md` 已存在的项目可继续使用；旧 `unknowns-and-decisions.md`、`refresh-policy.md` 和 `sdd-handoff.md` 仅保留一版只读兼容，不再是新流程门禁。
- 过程文档叙述默认使用中文；代码、命令、路径、API 字段和精确 schema 术语保留必要英文。
- PRD、proposal、design 和 plan 继续采用 compact-first：预计不超过 500 行与 30,000 字符时使用 small form，其他情况才按语义拆分。

## 验证

本版本通过插件全局校验，以及信息层、审查、CodeGraph、探索、产物布局与 Figma 视觉证据的 focused contract tests。
